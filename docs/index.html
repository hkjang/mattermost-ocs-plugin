Built for how Mattermost teams actually operate

The plugin is centered on documented OpenCode server APIs. That means the same integration can support general assistants, coding bots, project-aware task flows, and streaming responses without inventing a second execution model inside the Mattermost plugin.

Session-first architecture

Reuse OpenCode sessions per thread, DM, or channel policy so follow-up prompts stay grounded in the same conversation history.

Streaming that feels alive

Stream assistant output into one Mattermost post, including reasoning fade-out support for models that expose `` sections.

Admin-ready diagnostics

Check health, agents, providers, managed bots, and active sessions without leaving Mattermost administration screens.

Conversation and coding modes, one plugin

Conversation Mode

  • thread summaries and Q&A
  • support or drafting bots
  • default agent and model selection per bot
  • context-aware replies for channels, threads, and DMs

Coding Mode

  • OpenCode project and file API integration
  • coding task cards inside Mattermost
  • approved command execution through OpenCode sessions
  • session diff visibility for developer workflows

OpenCode-native by design

Coding workflows in this plugin are intentionally limited to what can be represented through OpenCode server APIs. Project context, file content, search, VCS data, shell execution, command execution, and session diffs all go through OpenCode instead of hidden local plugin behavior.
